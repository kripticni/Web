flexbox

a flexbox is a row/column layout for responsive container-based elements.

align-content how each line is aligned, if only applies if there are multipul lines as well as flex-wrap: wrap. options:
  • stretch, each line fills the space.
  • flex-start, only the space that is needed.
  • flex-end, only the space that is needed, but then also moved to the bottom of the flexbox
  • center, only the space that is needed, but then centered.
  • space-between, only the space that is needed, the rest goes between.
  • space-around, only the space that is needed, the rest goes between above and below.
align-items defines how to align to the cross axis, determined by flex-direction. options:
  • flex-start, align at start of the cross axis
  • flex-end, align at end of cross axis
  • center, center of the cross axis
  • baseline, aligns so that the items are above the baseline of the cross axis
  • stretch, fill up the whole space of the cross axis
align-self exactly like align-items, but only for a single item, it has the same options and is put inside the element you want to alter.
flex-basis defines the initial size of the flexbox item. options:
  • auto, sized based on its content
  • px/em/rem, based on pixels, also wrapped
flex-direction defines the cross axis of the flexbox. options:
  • row, horizontal, left to right
  • row-reverse, horizontal, right to left
  • column, vertical, top down
  • column-reverse, vertical, bottom up
flex-wrap single line or multiple line items. options:
  • nowrap, elements must be on a single line
  • wrap, elements can be on multiple lines if needed
  • wrap-reverse, any new line is above the original one
flex-flow shorthand for flex-direction and flex-wrap.
flex-grow how much to grow if there is available space, the option is the coefficient k of how much that exact element should take up which gets calculated into precentages.
flex-shrink how much to shrink if there isn't enough space, the option is also the coefficient k.
justify-content how to align according to the main axis. options:
  • flex-start, to the start of the container's main axis
  • flex-end, to the end of the container's main axis
  • center, centered along the main axis
  • space-between, remaining space is between the elements
  • space-around, remaining space is before between and after the elements
order the number that says where in the item list should the item be, it can be any large number
gap shorthand for row-gap and column-gap, can also be used for multi-column layouts and grids.
row-gap
column-gap a simple gap in length between columns or grid/table elements.

grid

how to make a grid with css

grid-area short for row-start, column-start, row-end, column-end, or for naming items. options:
  • auto, automatically sets for items
  • main, can be used to give items a name
grid-auto-columns defines the size of columns that are not inside of grid-template-columns or grid-template-areas, can be auto or pixels/em/rem.
grid-auto-rows defines the size of rows that are not inside of grid-template-rows or grid-template-areas, can be auto or pixels/em/rem.
grid-auto-flow defines the position of auto generated items. options
  • row, adds to its own row
  • column, adds to its own column
grid-column-end the column end position of a grid item, before which column to end.
grid-row-end the row end position of a grid item, before which row to end.
grid-row-gap defines the gap between rows, can be pixel/em/rem.
grid-column-gap defines the gap between columns, can be pixel/em/rem.
grid-gap shorthand for grid-row-gap and grid-column-gap.
grid-template-areas defines areas within a grid container, they can be referenced when placing a grid item. It can be none, or a list of strings which represent which cell is what.
grid-template-columns specify the width of each column in pixels/em/rem or auto.
grid-template-rows specify the width of each row in pixels/em/rem or auto.
grid-template a shorthand for grid-template-rows, grid-template-columns and grid-template-area.
grid a shorthand for grid-template-rows, grid-template-columns, grid-template-areas, grid-auto-rows, grid-auto-columns, and grid-auto-flow.
justify-items how to align items inside a grid:
  • legacy, legacy HTML center
  • normal, depends on layout similar to stretch
  • stretch, stretches to fill grid cell if width is not set
  • start
  • end
  • left
  • right
  • center
  • overflow-alignment, safe sets to start on overflow, unsafe does nothing
  • baseline alignment, how to align to the basement of the parent
justify-self how to align a gird item within its grid cell in the inline direction.

@

at-rules, instructions for the preprocessor

@charset the encoding of the .css file itself, it tell the browser how to interpret the bytes, its implied that its ASCII by default but with this you can change it to unicode. its expected at the top of the file and if there is multiple they are ignored.
@container define styles for other elements depending on the container's size or styles, following aspects can be used in containerquery
  • aspect-ratio
  • block-size
  • height
  • inline-size
  • orientation
  • width
@counter-style lets you create a custom style, the look of numbers in a counter using the attributes:
  • countername
  • system, cyclic, numeric, alphabetic, symbolic, fixed, for additive the additive-symbols is required
  • symbols
  • additive-symbols
  • negative, prepend or append if value is negative
  • prefix
  • suffix
  • range, for cycle
  • pad
  • speak-as, how speech synthesizers should read out the counter style
  • fallback, defines the fallback if the system is unable to construct the marker
@import import another css file into the current file, can use a string or url, must be at the top but after charset and layer, it can also be media dependant which is very useful.
@keyframes make an animation sequence by defining css styles for points along the sequence. within in:
  • name, required
  • keyframes-selector, %, from(%), to(%)
  • css-styles, one or more CSS properties
@layer first set the order of which classes are at which layers of importance, then when it applies over multiple tags the highest one will be chosen. a layer without a unique name is anonymous the order is the order of declaration, one with a unique name is called a named layer and can be ordered by using the @layer again.
@media media query for different widths/heights of the viewport/device, orientation and resolution. Supports logical not (inversion), and (combining media type with media features), only (prevents older browsers from applying styles they dont support).
@media not|only mediatype and (mediafeature and|or|not mediafeature) { ... }
they are used to create responsive websites.
  • media-types: all, print(ers), screen
  • media-features: any-hover, any-pointer, aspect-ratio, color, color-gamut, color-index, device-posture, display-mode, dynamic-range, forced-colors, grid, height, hover, inverted-colors, monochrome, orientation, overflow-block, overflow-inline, pointer, prefers-color-scheme, prefers-contrast, prefers-reduced-data, prefers-reduced-motion, prefers-reduced-transparency, resolution, scan, scripting, shape, update, video-dynamic-range, width
@namespace used to define an XML namespace to be used in the stylesheet, it must follow all charset and import rules, and precede all other rules. Can define a namespace prefix and need a url, like @namespace prefix url().
@page customizes the dimension, orientation and margins of printed pages. properties:
  • margin: value
  • page-orientation: value
  • size: value
@property allows for creating custom css properties, runs data type checking, constraining, default values and inheritableness:
  • syntax: data type
  • inherits: true or false
  • initial-value: data type value
@scope allows you to select elements in the DOM precisely without overly-specific selectors. used as:
  • @scope (scope root) { ... }
  • @scope (scope root) to (scope limit)
@starting-style define default element starting styles before the element gets its first style update, used for smooth entry and exit transitions:
  • @starting-style { ... }
@supports tests if a browser supports a certain css feature, used to override classes if a browser supports the feature like a conditional statement.
  • @supports (display: grid) { ... }

overflow

usually not good in computer science

overflow what happens if a content overflows an element's box:
  • visible
  • hidden, can be scrolled to
  • clip, forbids scrolling, the rest of the content is invisible
  • scroll, clipped but a scroll-bar is added to see the rest
  • auto, on overflow, adds a scroll-bar
disables the scroll anchoring of a browser, its a feature that prevents a viewable area that is scrolled into focus to move when new content is loaded above, highly situational:
  • auto, enabled
  • none, disabled
overflow-wrap can the browser break lines with long words if they overflow the container, usually good:
  • normal, long words willl not break
  • anywhere, break on overflow
  • break-word, break on overflow
overflow-x
overflow-y property specifies whether to clip the content, add a scrollbar or display overflow of a block-level element when it overflows left/right/top/bottom edges and how:
  • visible, can be rendered outside the edges
  • hidden, clipped no scroll
  • scroll, clipped with scroll
  • auto, scroll for overflowing boxes

perspective give a 3D positioned element some perspective, how far the object is away from the user, length or none.
perspective-origin defines from which position the user is looking at the 3D element. It applies to the child elements, used with perpsective. Takes two precentages or two bottom/top/center/left/right.

floating

all about out of place flying elements

float where the element should float, absolutely positioned elements ignore float and elements next to it will flow around it. options:
  • none
  • left
  • right
clear below which floating elements should this element be pushed:
  • none
  • left
  • right
  • both
clearfix hack a floating element taller than its container will overflow, to fix this use:
.clearfix::after {
  content: "";
  clear: both;
  display: table;
}
shape-outside tells the other elements how to go around the floating element. ways:
  • margin-box
  • border-box
  • padding-box
  • content-box
  • basic-shape, inset(), circle(), ellipse() or polygon()

masking and clipping

keep things hidden

clip (deprecated) deprecated and propertly replaced by clip-path, Clips an absolutely positioned element.
clip-path clip an element to a shape or SVG source. options:
  • clip-source, url to a SVG clippath element
  • basic-shape, circle(), ellipse(), polygon() or inset()
  • margin-box
  • border-box
  • padding-box
  • content-box
  • fill-box, object bounding box
  • stroke-box
  • view-box, svg viewport
  • none
mask shorthand for mask-image, mask-mode, mask-repeat, mask-position, mask-size and mask-composite
mask-clip painted content is clipped to:
  • border-box
  • content-box
  • padding-box
  • fill-box
  • stroke-box
  • view-box
  • no-clip
  • border/padding/content box
  • text
mask-composite specifies a compositing operation used on mask layer with the mask layers below it:
  • add
  • subtract
  • intersect
  • exclude
mask-image specifies which image to take, url or none.
mask-mode chooses between luminance and alpha mask (by default its match-source).
specifies the the origin of the mask box
  • border-box
  • content-box
  • padding-box
  • fill-box
  • stroke-box
  • view-box
mask-position set position to the mask layer image:
  • twice left/right/center center/top/bottom
  • % %
  • xpos ypos
mask-size
mask-height
mask-width specify the size of the mask layer image:
  • auto
  • size, length
  • contain, scale to fit inside
  • cover, scale to cover
mask-type specify if a svg mask element is treated as a luminance mask or as an alpha mask.
object-fit how a img or video should be resized to fit its container:
  • fill
  • contain, maintain ascept ratio
  • cover, fill the entire element while keeping the aspect ratio
  • none
  • scale-down
object-position how to position with object-fit with x/y coordinates inside its own content box:
  • position
  • initial
  • inherit
offset shorthand for offset-anchor, offset-distance, offset-path, offset-position and offset-rotate properties.
offset-anchor specifies the point on an element that is fixed to the path it is animated/placed along either left/right/center bottom/top/center or xpos, ypos or x%, y%.
offset-distance specifies the position along a path where an animated element is placed.
offset-path specifies a path for the element to be animated/placed along. make with:
  • path(), an svg path
  • ray(), using css ray function
  • url(), to an svg file
  • basic-shapem inset(), circle(), ellipse() or polygon()
  • coord-box
offset-position specifies the initial position of an element along a path.
offset-rotate specifies the rotation of a placed/animated element along a path, takes an angle, can be reverse or auto.

column

columnar
stuff

column-count specifies the number of columns to divide an element into, for example divide a paragraph into 3 columns of text.
column-fill how to fill the columns: balance, auto.
column-gap the gap between columns, takes length.
column-rule the border between columns, a shorthand for column-rule-color, column-rule-style and column-rule-width.
column-rule-color takes a color, if ommited is the color of the text.
column-rule-width the width of a column:
  • medium
  • thin
  • thick
  • length
column-span how many columns an element should span across. options:
  • column-width, minimum width for each column
  • column-count, maximum number of columns
  • auto, default

counters

when you count stuff

counter-increment increment or decrement the value of one or more css counters.
counter-reset creates and resets the value of one or more counters.
counter-set creates and sets the value of one or more counters.

display

pretty important

display how to display certain containers and elements:
  • inline, span, no height and width has effect
  • block, p, starts on a new line and takes up the entire width
  • contents, makes the container dissapear, making the child elements level up in DOM
  • flex, block level
  • grid, block level
  • inline-flex, inline level
  • inline-grid, inline level
  • inline-table
  • list-item, element behaves like li
  • run-in, block or inline depending on context
  • table, like table tag
  • table-caption, behave like caption tag
  • table-column-group, behave like colgroup tag
  • table-header-group, behave like thead element
  • table-footer-group, behave like tfoot tag
  • table-row-group, behave like tbody tag
  • table-cell, behave like td tag
  • table-column, behave like col tag
  • table-row, behave like tr element
  • none, element is completely removed

typography

all the text niceness you'll ever want

hanging-punctuation allows for text to be centered without its punctutations coming into the calculation, often makes it look better. options:
  • none
  • first, only the first line
  • last, only the last line
  • allow-end, if it doesnt fit before justification it can be outside
  • force-end, forces the punctuation to hang

stacking context

manage your z-index

isolation useful with background-blend-mode and mix-blend-mode does an element need to create a new stacking context:
  • auto, only if required by the properties
  • isolate, must be created anyways

transformations

change your 2D and 3D objects

transform applied a 2D or 3D transformation to elements:
  • none
  • matrix(6*n), 2d
  • matrix(16*n), 3d
  • translate(x, y)
  • translate3d(x, y, z)
  • translateX(x)
  • translateY(y)
  • translateZ(z)
  • scale(x, y)
  • scale3d(x, y, z)
  • scaleX(x)
  • scaleY(y)
  • scaleZ(z)
  • rotate(angle)
  • rotate3d(x, y, z, angle)
  • rotateX(angle)
  • rotateY(angle)
  • rotateZ(angle)
  • skew(ax, ay)
  • skewX(a)
  • skewY(a)
  • perspective(n)
transform-origin allows you to change the position of transformed elements, very similar to transfrom: translate(%) properties.
  • takes 2 of either:
  • left/center/right or length or %
  • top/center/bottom or length or %
  • if a 3rd value is given its the length for the Z axis
specifies how nexted elements are rendered in 3d space:
  • flat, no preserve
  • preserve-3d, preserves their 3d position
