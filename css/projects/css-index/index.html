flexbox

a flexbox is a row/column layout for responsive container-based elements.

align-content how each line is aligned, if only applies if there are multipul lines as well as flex-wrap: wrap. options:
  • stretch, each line fills the space.
  • flex-start, only the space that is needed.
  • flex-end, only the space that is needed, but then also moved to the bottom of the flexbox
  • center, only the space that is needed, but then centered.
  • space-between, only the space that is needed, the rest goes between.
  • space-around, only the space that is needed, the rest goes between above and below.
align-items defines how to align to the cross axis, determined by flex-direction. options:
  • flex-start, align at start of the cross axis
  • flex-end, align at end of cross axis
  • center, center of the cross axis
  • baseline, aligns so that the items are above the baseline of the cross axis
  • stretch, fill up the whole space of the cross axis
align-self exactly like align-items, but only for a single item, it has the same options and is put inside the element you want to alter.
flex-basis defines the initial size of the flexbox item. options:
  • auto, sized based on its content
  • px/em/rem, based on pixels, also wrapped
flex-direction defines the cross axis of the flexbox. options:
  • row, horizontal, left to right
  • row-reverse, horizontal, right to left
  • column, vertical, top down
  • column-reverse, vertical, bottom up
flex-wrap single line or multiple line items. options:
  • nowrap, elements must be on a single line
  • wrap, elements can be on multiple lines if needed
  • wrap-reverse, any new line is above the original one
flex-flow shorthand for flex-direction and flex-wrap.
flex-grow how much to grow if there is available space, the option is the coefficient k of how much that exact element should take up which gets calculated into precentages.
flex-shrink how much to shrink if there isn't enough space, the option is also the coefficient k.
justify-content how to align according to the main axis. options:
  • flex-start, to the start of the container's main axis
  • flex-end, to the end of the container's main axis
  • center, centered along the main axis
  • space-between, remaining space is between the elements
  • space-around, remaining space is before between and after the elements
order the number that says where in the item list should the item be, it can be any large number
gap shorthand for row-gap and column-gap, can also be used for multi-column layouts and grids.

grid

how to make a grid with css

grid-area short for row-start, column-start, row-end, column-end, or for naming items. options:
  • auto, automatically sets for items
  • main, can be used to give items a name
grid-auto-columns defines the size of columns that are not inside of grid-template-columns or grid-template-areas, can be auto or pixels/em/rem.
grid-auto-rows defines the size of rows that are not inside of grid-template-rows or grid-template-areas, can be auto or pixels/em/rem.
grid-auto-flow defines the position of auto generated items. options
  • row, adds to its own row
  • column, adds to its own column
grid-column-end the column end position of a grid item, before which column to end.
grid-row-end the row end position of a grid item, before which row to end.
grid-row-gap defines the gap between rows, can be pixel/em/rem.
grid-column-gap defines the gap between columns, can be pixel/em/rem.
grid-gap shorthand for grid-row-gap and grid-column-gap.
grid-template-areas defines areas within a grid container, they can be referenced when placing a grid item. It can be none, or a list of strings which represent which cell is what.
grid-template-columns specify the width of each column in pixels/em/rem or auto.
grid-template-rows specify the width of each row in pixels/em/rem or auto.
grid-template a shorthand for grid-template-rows, grid-template-columns and grid-template-area.
grid a shorthand for grid-template-rows, grid-template-columns, grid-template-areas, grid-auto-rows, grid-auto-columns, and grid-auto-flow.
justify-items how to align items inside a grid:
  • legacy, legacy HTML center
  • normal, depends on layout similar to stretch
  • stretch, stretches to fill grid cell if width is not set
  • start
  • end
  • left
  • right
  • center
  • overflow-alignment, safe sets to start on overflow, unsafe does nothing
  • baseline alignment, how to align to the basement of the parent
justify-self how to align a gird item within its grid cell in the inline direction.

used to color various html tags and more

accent-color sets the accent/primary color on specific tags, like radiobuttons, progress bars etc... options:
  • color, rgb, hex, html...

  • initial
  • inherit
background-color defines the background color of an element. options:
  • color, rgb, hex, html...
  • transparent, default
  • initial
  • inherit
caret-color specifies the color of the cursor in inputs, textareas, or anything editable.
color the foreground/text color of an element.
color-scheme indicates which operating system color scheme an element has. options are only (optional light/dark to say what to render with, and light dark says that it can be rendered with both.

animation

beautiful animations for your elements

animation shorthand for all animation-* properties.
animation-delay delay before the start of the animation.
animation-direction should the animation be played:
  • normal
  • reverse
  • alternate, forward then backward
  • alternate-reverse, first backwards, then cycles
  • initial
  • inherit
animation-duration how long does an animation last, can be:
  • time, ms or s
  • initial
  • inherit
animation-fill-mode specifies a style when the animation is not playing. options:
  • none
  • forwards, last keyframe
  • backwards, first keyframe
  • both, extends the property in both directions
  • initial
  • inherit
animation-iteration-count number of repetitions, infinite or a number.
animation-name specifies a name for the @keyframes animation.
animation-play-state is the animation playing or pause.
animation-timing-function decides which time function to use for the animation. options:
  • linear
  • ease, default, slow->fast->slow
  • ease-in, slow start
  • ease-out, slow end
  • ease-in-out, slow end and start
  • step-start, steps(1, start)
  • step-end, steps(1, end)
  • steps(int, step-position), the stepping function
  • cubic-bezier(n,n,n,n), cubic-bezier function, values are 0-1
  • initial
  • inherit

size

the sizing settings for elements

aspect-ratio specifies the preferred aspect ratio, given as a x / y
height determines the height, length or %.
inline-size size of an element in the inline direction, determined by writing-mode.
block-size size of an element in the block direction, determined by writing-mode.

writing-mode sets the writing-mode to block or inline.
block-size specifies the size of an element in the block direction, determined by writing-mode property, takes length.
direction doesn't change or affect writing mode but specifies the text/writing direction to ltr or rtl.

background

backgrounds to elements

background a shorthand to all background-* properties.
background-color defines the background color of an element. options:
  • color, rgb, hex, html...
  • transparent, default
  • initial
  • inherit
background-image a url for the background image.
background-position position can be a combination of two: left/right top/bottom/center.
background-position-x image position on x axis, left right center or px/em/rem.
background-position-y image position on y axis, left right center or px/em/rem.
background-size size of background image. options:
  • auto, default, original size
  • length, width and height
  • precentage, can be used for proportion
  • cover, resize the image to cover the entire container
  • contain, resize to make the image fully visible
  • initial
  • inherit
background-repeat how the background should or shouldn't be repeated. options:
  • repeat
  • repeat-x
  • repeat-y
  • no-repeat

  • round, repeated and squished to fill the space
  • intial
  • inherit
background-origin from where to start the background image. options:
  • padding-box, upper left corner of the padding edge
  • border-box, upper left corner of the border

  • inherit
background-clip how far should a background extend within an element

  • border-box, behind the border
  • content-box, edge of content box
  • initial
  • inherit
background-attachment whether the image scrolls with the page or not. options:
  • scroll, default
  • fixed, won't scroll
  • local, scroll with the element's content
  • initial
  • inherit
background-blend-mode ways to blend together layers of the background:

breaks

control the content flow

a page break a page is like sheet of paper, they matter when printing a page, or generating a pdf, a page break tells the browser where to start a new page.
a column break when text flows into multiple columns, tell the browser to move the content into the next column instead of continuing in the current one.
a region break regions are rare and obsolete, an old css 2012 idea that let you pour text into multiple divs, stop the content and continue in the next region box. this is not practical today.
break-after
break-before
break-inside whether a page, column or region break should happen after, before or inside a specific element. options:
  • auto
  • avoid
  • avoid-column
  • avoid-page
  • avoid-region

@

at-rules, instructions for the preprocessor

@charset the encoding of the .css file itself, it tell the browser how to interpret the bytes, its implied that its ASCII by default but with this you can change it to unicode. its expected at the top of the file and if there is multiple they are ignored.
@container define styles for other elements depending on the container's size or styles, following aspects can be used in containerquery
  • aspect-ratio
  • block-size
  • height
  • inline-size
  • orientation
  • width
@counter-style lets you create a custom style, the look of numbers in a counter using the attributes:
  • countername
  • system, cyclic, numeric, alphabetic, symbolic, fixed, for additive the additive-symbols is required
  • symbols
  • additive-symbols
  • negative, prepend or append if value is negative
  • prefix
  • suffix
  • range, for cycle
  • pad
  • speak-as, how speech synthesizers should read out the counter style
  • fallback, defines the fallback if the system is unable to construct the marker
@import import another css file into the current file, can use a string or url, must be at the top but after charset and layer, it can also be media dependant which is very useful.
@keyframes make an animation sequence by defining css styles for points along the sequence. within in:
  • name, required
  • keyframes-selector, %, from(%), to(%)
  • css-styles, one or more CSS properties
@layer first set the order of which classes are at which layers of importance, then when it applies over multiple tags the highest one will be chosen. a layer without a unique name is anonymous the order is the order of declaration, one with a unique name is called a named layer and can be ordered by using the @layer again.

pretty important

  • flex, block level
  • grid, block level
  • inline-flex, inline level
  • inline-grid, inline level
  • inline-table
  • list-item, element behaves like li

  • table-header-group, behave like thead element

  • table-row-group, behave like tbody tag

  • table-column, behave like col tag
