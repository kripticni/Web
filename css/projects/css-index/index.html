flexbox

a flexbox is a row/column layout for responsive container-based elements.

align-content how each line is aligned, if only applies if there are multipul lines as well as flex-wrap: wrap. options:
  • stretch, each line fills the space.
  • flex-start, only the space that is needed.
  • flex-end, only the space that is needed, but then also moved to the bottom of the flexbox
  • center, only the space that is needed, but then centered.
  • space-between, only the space that is needed, the rest goes between.
  • space-around, only the space that is needed, the rest goes between above and below.
align-items defines how to align to the cross axis, determined by flex-direction. options:
  • flex-start, align at start of the cross axis
  • flex-end, align at end of cross axis
  • center, center of the cross axis
  • baseline, aligns so that the items are above the baseline of the cross axis
  • stretch, fill up the whole space of the cross axis
align-self exactly like align-items, but only for a single item, it has the same options and is put inside the element you want to alter.
flex-basis defines the initial size of the flexbox item. options:
  • auto, sized based on its content
  • px/em/rem, based on pixels, also wrapped
flex-direction defines the cross axis of the flexbox. options:
  • row, horizontal, left to right
  • row-reverse, horizontal, right to left
  • column, vertical, top down
  • column-reverse, vertical, bottom up
flex-wrap single line or multiple line items. options:
  • nowrap, elements must be on a single line
  • wrap, elements can be on multiple lines if needed
  • wrap-reverse, any new line is above the original one
flex-flow shorthand for flex-direction and flex-wrap.
flex-grow how much to grow if there is available space, the option is the coefficient k of how much that exact element should take up which gets calculated into precentages.
flex-shrink how much to shrink if there isn't enough space, the option is also the coefficient k.
justify-content how to align according to the main axis. options:
  • flex-start, to the start of the container's main axis
  • flex-end, to the end of the container's main axis
  • center, centered along the main axis
  • space-between, remaining space is between the elements
  • space-around, remaining space is before between and after the elements
order the number that says where in the item list should the item be, it can be any large number

grid

how to make a grid with css

grid-area short for row-start, column-start, row-end, column-end, or for naming items. options:
  • auto, automatically sets for items
  • main, can be used to give items a name
grid-auto-columns defines the size of columns that are not inside of grid-template-columns or grid-template-areas, can be auto or pixels/em/rem.
grid-auto-rows defines the size of rows that are not inside of grid-template-rows or grid-template-areas, can be auto or pixels/em/rem.
grid-auto-flow defines the position of auto generated items. options
  • row, adds to its own row
  • column, adds to its own column
grid-column-end the column end position of a grid item, before which column to end.
grid-row-end the row end position of a grid item, before which row to end.
grid-row-gap defines the gap between rows, can be pixel/em/rem.
grid-column-gap defines the gap between columns, can be pixel/em/rem.
grid-gap shorthand for grid-row-gap and grid-column-gap.
grid-template-areas defines areas within a grid container, they can be referenced when placing a grid item. It can be none, or a list of strings which represent which cell is what.
grid-template-columns specify the width of each column in pixels/em/rem or auto.
grid-template-rows specify the width of each row in pixels/em/rem or auto.
grid-template a shorthand for grid-template-rows, grid-template-columns and grid-template-area.
grid a shorthand for grid-template-rows, grid-template-columns, grid-template-areas, grid-auto-rows, grid-auto-columns, and grid-auto-flow.
