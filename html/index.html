List of html elements

<!--...-->, used to mark a comment in html.

<!DOCTYPE html>, is at the start of the source indicating the type of document.

<html>, wraps the entire markup of the page. Its common to add lang="en" for SEO, screen reader support and spell-checking/translation.

<meta>, means metadata about the document its common to set charset="UTF-8" to be able to render all characters and viewport settings such as <meta name="viewport" content="width=device-width, initial-scale=1.0"> for mobile, and set a description and keywords for SEO.

<base>, specifies a base url for all future urls.

<script>, used for linking external javascript.

<noscript>, used for giving alternate content for users with no client-side script support.

<link>, the link tag is used for many functionalities, most commonly for linking with external css, you use it with the rel attribute and it support:

  • stylesheet
  • icon
  • preload
  • prefetch
  • dns-prefetch
  • preconnect
  • alternate
  • manifest
  • canonical
  • apple-touch-icon

<style>, used to set internal css into the html document.

<head>, only changes the display text of the page in the browser

<body>, the actual contents of the page.

<hgroup>, used to embed a heading and paragraph inside to show that they are realted.

<h1-6>, headings differing in size from biggest to smallest.

<p>, a basic paragraph.

<br>, puts a
line break.

<b>, simply bolds the said text.

<strong>, allows the text to be styled according to the defined style of strong (by default, only bold).

<i>, simply makes the text italic.

<em>, also allows the text to be styled according to the defined style of emphasis (by default, only italic).

  • <ul>, unordered list or bullet point list, has its list items.
  • <ol>, ordered list, has its list elements.
  • <li>, a list element.
<dl>
Used to make a list with descriptive list.
<dt>
Used to give a title for a descriptive list item.
<dd>
Used to give the description for said item.

<menu>, a unordered list with semantic meaning, it most commonly represents a list of actions a user can take.

<a>, used to create hyperlinks to either the same page or a new one and for downloading files. It has the following atributes and options for them:

  1. href, for the link or heading of the page.
  2. target, where the page should be opened, _blank, _self, _parent, _top.
  3. rel, relationship between the pages, nofollow, noopener.
  4. title, extra info showed on hover.
  5. download, tells the browser to download instead of opening as a page.

Explanation of all table tags
HTML element Meaning
<table> Opens the table element allowing for it to be created.
<thead> Defines what goes into the header of the table.
<tr> Defines what goes into the row of the table.
<tbody> Defines what goes into the body of the table.
<td> Defines what goes into the cell of the table.
<colgroup> Specifies a group of one or more columns for formatting.
<col> Used to apply styles to a column within colgroup.
<tfoot> The footer part of a table.

<hr>, puts a horizontal line.

<div>, used to group up content and takes the full width.

<span>, opposed to div is an inline container, easy to manipulate with css.

Legend for Form:

<form>, a section of the page that is sent to the server with input elements. Has parameters:

<fieldset>, a semantic container for grouping user input or form controls, also gives a style.

<legend>, defines the title/legend for the fieldset.

  • action, to which file to send the form.
  • method, with which request to send the data.

<embed>, used for external resources, but most already have their own dedicated tags so it isn't very common.

<object>, used for adding plugins to extend the default browser functionality but nowadays isn't supported in most browsers, with the param tag you can define the parameters for the said object.

<img>, a image tag that is self closing uses:

  • src, the source path of the image.
  • alt, the path of the alternative image if source isn't found.
  • width, the pixel width of the image.
  • height, the pixel height of the image.
  • they can also we wrapped around anchor tags.

<figure>, specifies self-contained content, more useful with css.

<source>, from which file/url to get the source to be displayed accross multiple tags (audio, video, picture...), also multiple can be defined and the browser will use the first one that it supports, to set a source use the srcset attribute (for picture). Otherwise use src when it only needs to display one file (audio, video...). To query for conditions use the media attribute.

<picture>, used to define a window in which the first source tag that fits the current condition of the page is inputted in, if there is no source tag that fits, instead an img tag is used (it must always be at the bottom).

<svg>, used to create svg elements, along with tags like circle, ellipse and etc.

<audio>, used to embed sound content. Options:

  • controls, shows the default controls
  • autoplay, autoplays
  • loop, repeats when it ends
  • muted, starts off as muted
  • preload, how much to load upfront, can be none/metadata/auto.
  • src, audio file to play
  • crossorigin, if loading audio from another domain, can be anonymous/use-credentials

<video>, used to show video, can be used with the source tag to allow the browser to choose which one is best but also just the src attribute if it needs to show only one, options:

  • controls, shows the default controls
  • autoplay, autoplays
  • loop, repeats when it ends
  • muted, starts off as muted
  • preload, how much to load upfront, can be none/metadata/auto.
  • src, audio file to play
  • crossorigin, if loading audio from another domain, can be anonymous/use-credentials
  • poster, image before the video starts
  • width/height, the width and height of the video.
  • playsinline, prevents full-screen on mobile

<track>, used to add subtitles/captions to audio or video, uses the vtt format.

<canvas>, creates a canvas in which you can draw with javascript.

<map>, used to set coordinates for clickable areas within an img tag, the img tag must have the attribute usemap with #mapname and the map must have the name attribute. Within the map tag, you need to use the area tag with the shape and coords attributes to define the clickable areas and then href it to the correct page.

<blockquote>, used to quote some text, with the cite parameter which references who you are citing.

<abbr>, used to abbreviate text with the title parameter in html.

There is lots of semantic tags in html, used for specific parts of the page such as header, nav, section, aside, article, details, dialog, footer, main, search, summary, data.

<dialog>, shows a dialog window, if used with open then its shown, and role can be used for accesibility.

<data>, gives machine readable translations for text.

<address>, used to mark the contact info of the

author/owner
.

<area>, defines areas within an image with clickable links/interactions.

<bdi>, bi-directional isolation, doesnt allow the text inside to change direction of the entire paragraph, example: إيان.

<bdo>, overrides the text direction.

<cite>, defines the title of a work.
<code>, defines a piece of code.

<del>, crosses out the text as deleted from a document, its <s> with semantic meaning.

<dfn>, clarifies which word will be defined within the paragraph.

<ins>, underlines the text, semantic meaning of <u>.

<kbd>, tells the user that the following text is actually keyboard input.

<mark>, highlights the given text.

<meter>, a simple progress bar, example: 5/10.

<for>, used for semantically labeling the elements to which id they use.

<id>, tells the element which css id to use.

<class>, tells the element which css classes to use.

<progress>, shows the progress of a task like downloading, example: 50%.

<pre>, allows for the usage of preformatted text (html doesn't format it).

<q>, a small quotation with quotation marks.

<ruby>, defines ruby annotation for East Asian typography, example: 漢 ㄏㄢˋ .

<rt>, explains the pronunciation of characters for East Asian typography. Used inside of ruby to say what text should be written.

<rp>, defines what to show in browsers that don't allow for ruby notation, example: 漢 (ㄏㄢˋ)

<s>, strike through text.

<samp>, sample output of a program.

<small>, gives small text, will probably be deprecated.

<sub>, subscripts the text.

<sup>, supscripts the text.

<template>, hidden html content that you can later display with javascript on demand. example: let clon = temp.content.cloneNode(true);

<time>, puts time/datetime, 10:10 / Unix Epoch, basically only semantic meaning, it isn't supported in most browsers.

<u>, underlines the text.

<var>, formats the text as if it were a variable (mathematical).

<wbr>, where is it okay to add a linebreak, for example for very long words.

<iframe>, creates an inline frame in which you can embed another document, use css to style it.
